# Bug Bounty Research Journal

Public, sanitized notes from an ongoing responsible-disclosure workflow focused on open-source security, AI/ML systems, model-file vulnerabilities, and practical bounty triage.

This repository is intentionally not a dump of private reports. Active findings, exploit details, private program URLs, proof-of-concept files, tokens, and unpublished vulnerability chains stay private until disclosure is authorized or the issue is already public.

## What is here

- Methodology for selecting targets and avoiding low-signal duplicate work.
- Platform notes for bounty sources we evaluated.
- A high-level research timeline with sensitive details removed.
- Responsible disclosure rules we follow.
- Placeholders for case studies that can be published after coordinated disclosure.

## Current Focus

- AI/ML open-source security programs.
- Model file vulnerability research.
- Auth, access-control, SSRF, artifact-handling, and scanner-bypass classes.
- Smaller paid open-source issues only when the fix is clear, low-noise, and not already saturated.

## Disclosure Policy

We do not publish enough detail to reproduce an active vulnerability before the maintainer, platform, or program has cleared disclosure.

For pending findings, public notes are limited to broad category, target family, status, and lessons learned. Full technical write-ups are published only after remediation, public advisory, CVE publication, or explicit approval.

## Status

This repository was created as a public record of process, not as a live exploit archive.
